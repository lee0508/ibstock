# 2026-05-10 검색품질개선 하이브리드 리랭킹

## 구현 목적
- HANDOFF.md의 다음 우선 작업 3번(검색 품질 개선)을 3시간 작업 단위로 진행.
- 기존 SQLite exact match + FAISS 결과를 유지하면서 상위 노출 정확도를 높이는 리랭킹 추가.

## 변경 파일
- backend/app/api/stocks.py

## 구현 내용
1. 쿼리 토큰화 함수 추가
- `_query_tokens(query: str)`
- 구분자(`/`, `,`) 정규화 후 토큰 분리.
- 전체 lower 문자열도 토큰에 포함해 부분 일치 탐색 강화.

2. 재정렬 점수 함수 추가
- `_rerank_score(item, tokens)`
- 필드별 가중치:
  - 종목(`stocks`) 일치: +4
  - 기관(`ib_names`) 일치: +3
  - 제목(`title`) 일치: +2
  - 섹터(`sectors`) 일치: +2
  - 본문(`description/raw_text`) 일치: +1
- 기존 벡터 점수(`score`)는 `0.1` 배로 반영.

3. 검색 결과 병합 후 정렬 방식 개선
- `resolve_search_items()`에서 exact + FAISS 병합 결과에 점수 부여.
- 중복 제거 후 `score` 내림차순 정렬.
- fallback(`search_stock_mentions`) 결과도 동일한 리랭킹 적용.
