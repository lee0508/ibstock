# 2026-05-10 로컬실기동Query검증자동화구현

## 목적
- HANDOFF 우선순위 1/2(로컬 기동 검증, query 생성형 응답 실검증)를 반복 가능한 방식으로 수행.
- 검증 결과를 Markdown 파일로 자동 기록.

## 변경 파일
- scripts/verify_local_demo.py
- LOCAL_DEMO_RUNBOOK.md
- 2026-05-10_로컬실기동및Query검증.md (실행 결과 리포트)

## 구현 내용
1. 검증 스크립트 추가 (`verify_local_demo.py`)
- `/api/health` 호출 결과 확인
- `/api/stock/query` 3개 질의(대한전선, JP모건, 원자력) 호출
- 응답 성공/실패, item_count, answer preview 수집
- 결과를 `2026-05-10_로컬실기동및Query검증.md`로 출력
- health/query 실패 시 종료코드 1 반환

2. Runbook 반영
- 시연 순서에 자동 검증 명령(`python -X utf8 scripts\\verify_local_demo.py`) 추가

## 테스트
1. 컴파일 테스트
- 명령: `python3 -m compileall scripts`
- 결과: OK

2. 실환경 검증 실행
- 명령: `python3 scripts/verify_local_demo.py`
- 결과: health failed (현재 세션에서 웹서버 미기동)
- 리포트: `2026-05-10_로컬실기동및Query검증.md` 생성

3. 모의 서버 스모크 테스트
- 검증 스크립트에 대한 자체 동작 확인용 mock API 서버 실행 후 테스트
- 결과: all checks passed, exit_code=0
- 리포트: `2026-05-10_로컬실기동및Query검증_mock.md` 생성

## 메모
- 실제 PASS 리포트는 Windows PowerShell에서 `run_ollama_local.ps1` + `run_local_demo.ps1` 기동 후 동일 명령으로 재생성 가능.
