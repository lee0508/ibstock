# 2026-05-10 헬스체크 Ollama 연결 검증 추가

## 작업 배경
- HANDOFF 우선순위 1/2(로컬 기동 검증, query 생성형 응답 실검증)을 빠르게 확인하기 위해 health API에 Ollama 실연결 상태를 포함.

## 변경 파일
- backend/app/api/health.py

## 구현 내용
1. `/api/health`에 Ollama 연결 점검 로직 추가
- `GET {OLLAMA_HOST}/api/tags` 호출
- 응답 성공 시:
  - `ollama.reachable = true`
  - `ollama.model_count = len(models)`
- 실패 시:
  - `ollama.reachable = false`
  - `ollama.error`에 예외 문자열 기록

2. health 응답 구조 확장
- 기존 `ollama_host` 단일 문자열 대신 아래 구조로 반환:
  - `ollama.host`
  - `ollama.reachable`
  - `ollama.model_count`
  - `ollama.error`(실패 시)

## 테스트
1. 컴파일 테스트
- 명령: `python3 -m compileall backend`
- 결과: OK

2. 단위 스모크 테스트(스텁 기반)
- fastapi/httpx 미설치 환경을 고려해 스텁으로 health 함수 호출 검증
- 검증 항목:
  - `status == "ok"`
  - `ollama.reachable == true`
  - `ollama.model_count == 1`
- 결과: `health-unit-smoke: OK`

## 기대 효과
- 데모 시작 전에 `/api/health` 한 번으로 Ollama 준비 상태를 즉시 확인 가능.
- `/api/stock/query` 실패 원인(모델 미기동/연결 불가)을 빠르게 분리 가능.
