# 2026-05-10 프리플라이트체크자동화

## 목적
- HANDOFF 우선순위 1/2(로컬 기동 검증, 생성형 응답 실검증) 준비 시간을 단축.
- 데모 시작 전에 필수 산출물/DB/Ollama 상태를 한 번에 점검.

## 변경 파일
- scripts/preflight_check.py
- scripts/run_preflight.ps1
- LOCAL_DEMO_RUNBOOK.md

## 구현 내용
1. Python 프리플라이트 스크립트 추가 (`scripts/preflight_check.py`)
- 점검 항목
  - raw/processed 데이터 파일 존재
  - SQLite DB 파일 존재
  - FAISS 인덱스/메타/빌드정보 파일 존재
  - DB `shorts_videos` row count > 0
  - Ollama `/api/tags` 연결 가능 여부
- 출력 형식
  - `PASS` / `WARN` / `FAIL`
  - 최종 `RESULT` 출력
- 종료 코드
  - 필수 항목 실패 시 `1`
  - Ollama 미연결은 기본 `WARN` (종료코드 0)
  - `--strict-ollama` 옵션 사용 시 Ollama 미연결도 실패 처리

2. PowerShell 실행 래퍼 추가 (`scripts/run_preflight.ps1`)
- 명령: `python -X utf8 scripts\preflight_check.py --ollama-host http://127.0.0.1:11434`

3. Runbook 반영 (`LOCAL_DEMO_RUNBOOK.md`)
- 웹서버 실행 전 preflight 실행 단계 추가
- preflight 단독 실행 방법과 strict 옵션 예시 추가

## 테스트
1. 컴파일 테스트
- 명령: `python3 -m compileall backend scripts`
- 결과: OK

2. 실행 테스트
- 명령: `python3 scripts/preflight_check.py`
- 결과: OK (`RESULT: OK (ollama warning)`)
- 관찰: Ollama 미기동 상태에서 `WARN` 출력, 필수 데이터/DB/인덱스는 모두 PASS

## 기대 효과
- 데모 시작 전 실패 원인을 즉시 분리 가능(파일/DB/인덱스 vs Ollama).
- `/api/stock/query` 실검증 전에 환경 준비도를 빠르게 확인 가능.
