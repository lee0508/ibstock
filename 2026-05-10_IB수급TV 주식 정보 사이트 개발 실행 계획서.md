### 6-2. SQLite (경량 관계형 DB) — 신규 추가

영상 메타데이터 및 수급 이력 관리용 SQLite DB를 추가한다.

**테이블 구조**:

```sql
-- 영상 정보 테이블
CREATE TABLE shorts_videos (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  video_id TEXT UNIQUE NOT NULL,
  title TEXT NOT NULL,
  published_at DATE,
  views INTEGER,
  category TEXT,
  collected_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 종목 매핑 테이블
CREATE TABLE stock_mentions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  video_id TEXT,
  stock_name TEXT,
  sector TEXT,
  ib_source TEXT,
  mention_type TEXT,  -- 'buy', 'watch', 'sell'
  FOREIGN KEY (video_id) REFERENCES shorts_videos(video_id)
);

-- IB기관 수급 테이블
CREATE TABLE ib_supply (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  video_id TEXT,
  ib_name TEXT,       -- 골드만삭스, JP모건 등
  stock_name TEXT,
  action TEXT,        -- '매수', '매집', '순매수'
  date DATE,
  FOREIGN KEY (video_id) REFERENCES shorts_videos(video_id)
);

-- 섹터 분류 테이블
CREATE TABLE sectors (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT UNIQUE NOT NULL,
  description TEXT,
  keywords TEXT      -- JSON 배열
);
```

```bash
# 프로젝트 디렉터리 생성
mkdir -p ~/stock-rag/{backend,frontend,data/{raw,processed},scripts}
cd ~/stock-rag

# 가상환경 생성
python3 -m venv venv
source venv/bin/activate
```

**작업 목록**:
1. yt-dlp로 @Grandpa_Investor_Ib 채널 전체 Shorts 메타데이터 수집
2. 제목·설명·태그 텍스트 전처리 (종목명, IB기관명, 섹터 추출)
3. SQLite DB 스키마 생성 및 데이터 적재
4. FAISS 인덱싱 파이프라인 실행 (주식 데이터 적용)

**수집 스크립트 예시** (`scripts/collect_shorts.py`):
```python
import yt_dlp
import json
import sqlite3
from datetime import datetime

CHANNEL_URL = "https://www.youtube.com/@Grandpa_Investor_Ib/shorts"

def collect_shorts():
    ydl_opts = {
        'skip_download': True,
        'extract_flat': True,
        'quiet': True,
    }
    with yt_dlp.YoutubeDL(ydl_opts) as ydl:
        result = ydl.extract_info(CHANNEL_URL, download=False)
        videos = result.get('entries', [])
    return videos

if __name__ == "__main__":
    videos = collect_shorts()
    with open('data/raw/shorts_list.json', 'w', encoding='utf-8') as f:
        json.dump(videos, f, ensure_ascii=False, indent=2)
    print(f"수집 완료: {len(videos)}개")
```

### Phase 4 — 자동화 스케줄러 (4주차)

```python
# scripts/scheduler.py
from apscheduler.schedulers.blocking import BlockingScheduler

scheduler = BlockingScheduler(timezone='Asia/Seoul')

@scheduler.scheduled_job('cron', hour=6, minute=0)
def daily_collect():
    """매일 오전 6시 새 Shorts 수집 및 인덱스 갱신"""
    collect_shorts()
    preprocess_data()
    rebuild_faiss_index()
    print(f"[{datetime.now()}] 자동 수집 완료")

scheduler.start()
```

```bash
# systemd 서비스 등록
sudo nano /etc/systemd/system/stock-scheduler.service
```

```ini

Restart=always

[Install]

```

## 11. 주의사항 및 제한

- **YouTube 크롤링 정책**: yt-dlp는 오픈소스이나 YouTube ToS에 따라 상업적 사용 시 제한 가능. 개인/비공개 내부 사용 권장
- **YouTube Data API**: 무료 할당량 10,000 units/일. Shorts 메타데이터 수집은 할당량 내 처리 가능
- **FAISS CPU 버전**: GPU 없이 동작하나, 대용량 인덱스(100MB+)는 검색 속도 감소 가능 → 인덱스 분리 권장
- **Ollama 한국어 모델**: qwen3:latest는 한국어 지원 양호. EXAONE은 LG AI 연구원 제공 무료 모델
